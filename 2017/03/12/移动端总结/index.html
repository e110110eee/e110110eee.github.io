这周工作相对还是很轻松的。主要还是适应新的项目，对于项目整体已经比较了解了。主要做的主要是人脸识别的前端实现的模块。还有就是对项目前端页面的优化。

前端页面使用了一个叫amazeUI的框架，这个框架是我大中国人发明的。很大程度上借鉴了bootstrap，分离了一部分出来。可能这个更加轻便。更加实用。

百度地图

前端页面加入了一个在百度地图上显示marker的功能。在还没开始做之前以为很难。其实也不过如此。把这一部分组合成了一个组件。之后就可以直接拿来用了。 简单说一下主要的思路吧。

先创建一个百度地图的实例

map = new BMap.Map("map");

//第1步：设置地图中心点，

var point = new BMap.Point(lng, lat);

//第2步：初始化地图,设置中心点坐标和地图级别。 

map.centerAndZoom(point, 20);

//设置需要的组件
//第3步：启用滚轮放大缩小 

map.enableScrollWheelZoom(true);

//第4步：向地图中添加缩放控件 

var ctrlNav = new window.BMap.NavigationControl({  
    anchor: BMAP_ANCHOR_TOP_LEFT,  
    type: BMAP_NAVIGATION_CONTROL_LARGE  
});  
map.addControl(ctrlNav);

//第5步：向地图中添加缩略图控件 

var ctrlOve = new window.BMap.OverviewMapControl({  
    anchor: BMAP_ANCHOR_BOTTOM_RIGHT,  
    isOpen: 1  
});  
map.addControl(ctrlOve);

//第6步：向地图中添加比例尺控件 

var ctrlSca = new window.BMap.ScaleControl({  
    anchor: BMAP_ANCHOR_BOTTOM_LEFT  
});  
map.addControl(ctrlSca);

//开始绘制点markerArr是放置点的数组 ，绘制使用new BMap.Marker(point, { icon: myIcon })

for (var i = 0; i < markerArr.length; i++) {  
    var p0 = markerArr[i].point.split(",")[0];  
    var p1 = markerArr[i].point.split(",")[1];  
    var maker = addMarker(new BMap.Point(p0, p1), i);  
}  
//addMarker
// 添加标注  
function addMarker(point, index) {  
    var myIcon = new BMap.Icon("http://api.map.baidu.com/img/markers.png",  
        new BMap.Size(23, 25), {  
            offset: new BMap.Size(10, 25),  
            imageOffset: new BMap.Size(0, 0 - index * 25)  
        });  
   var marker = new BMap.Marker(point, { icon: myIcon }); 
    map.addOverlay(marker);  
    return marker;  
}

最后我再说一点，创建百度地图的实例的时候，最好是马上初始化。绘制需要的地图，要不然会出现地图绘制不完整的问题。如果图片的中心使用GPS定位的话。地图绘制也会因为这个问题出现一些偏差。

人脸识别

言归正传，说点有趣的东西吧，这周主要做的就是人脸识别。分成两个部分，一个是身份证的上传。另一个是人脸比对。先从上传身份证开始说起吧。使用的是face++来识别身份证。但是这有一个问题，检测的时候需要让图片都是竖直的才行。当然一开始的思想就是判断手机拍摄的图片的宽高来判断是否需要让图片旋转。图片旋转我们可以使用CSS3的transform：rotate（deg）来实现。判断宽高比也是比较容易的。直接拿到拍摄照片的宽高就好了。
在安卓的手机上测试拍照的时候还是很好的。众所周知苹果手机就喜欢搞事情的。问题就出在： 苹果手机拍摄的图片不管是横屏拍摄还是竖屏拍摄都是高比宽要大。 。而且苹果手机在图片绘制时候有一些问题。我们就采取办法：指引用户都使用横屏拍摄。
然后就使用到了一个JQ模板框的插件。感觉挺酷炫的。叫layer.js

layer.js

这个模板框感觉很酷，比较方便的制作一些提示框，警告框什么的。自己也做了几个以备不时之需传送门
初始化如下：

layer.open({
  title: 'hello layer'
  ,content: '可以填写任意的layer代码'
});

EXif.js

原本以为这样指引就可以解决了的。但是苹果又淘气了。苹果手机有时候在横屏拍照的时候有时候并不会自己旋转过来。如果把手机垂直地面横屏拍摄。这样就能感应到自动旋转。也许是苹果手机特有的设计。认为你会拍一些横屏的东西。 等我有钱第一件事就是买苹果手机 。 这样就还会有些问题。觉得没有什么方法的时候。发现之前大神写的代码里面有一个叫EXIF.js的东西。赶紧google了一下。发现这个插件也有点6。只能够拿到苹果手机的拍摄的方向。其他手机任意拍摄方向返回的都是1或者是“undefined”。这样就能判断苹果手机的图片拍摄方向了。
除了得到拍摄方向，里面还有一个参数比较感兴趣的是拍摄照片的时间。这个在业务场景也是很有用的。可以判断图片是否是刚拍的，然后判断图片的真伪度。感觉这个插件比较6，还能压缩图片，获取图片其他很多信息。还在研究这个插件的源码。

总结

这一周感觉收获还是很多的。认识很多同事，在解决问题的时候又有了新的看法。当找到解决办法的时候，应该先想清楚是否还有其他的办法去解决问题。而不是马上去实现。因为一开始想到的不一定是最好的。
